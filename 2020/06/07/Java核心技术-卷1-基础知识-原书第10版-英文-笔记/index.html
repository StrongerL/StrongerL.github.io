Java简介

Java名称由来

Java起初的名称是Oak（起这个名字的原因可能是项目的领导者之一James Gosling办公室外有一棵橡树），后来发现该名称已被另一种语言使用，就改名为了Java。一种关于命名为Java的传闻是在命名时项目组中的一人在喝名为Java的咖啡，因此起名为Java。有人为此专门采访了项目组的成员，但是大家基本上都无法详细说出命名为Java的真正原因，它出现在这次命名会议中，作为多个候选名字中的一个，最终获得通过，该采访可以在这篇文章中看到。

Java发展历程

1991年，Sun公司中由Patrick Naughton和James Gosling领导的一个工程师组想要为消费者设备（如有线电视开关盒）开发一种语言，这个项目被称为Green。

1992年，Green项目发布了第一款产品，称作“*7”，然而他们的产品找不到购买的厂商。

后来该项目改名为“First Person, Inc”，由于找不到购买者，该项目在1994年解散。

与此同时，万维网也迅猛地发展着。为了展示Java的功能，Java语言开发者开发了一个浏览器——HotJava，这激发了使用Java的热潮。

Sun公司在1996年发布了第一版的Java，之后陆续发布了更多的版本。直到2009年Sun公司被Oracle公司收购，此后，Java便由Oracle公司进行维护，截止到Java8的版本发布历史可以见下图。

JDK相关概念

JDK与JRE区别

JDK是Java Development Kit（Java开发工具集）的缩写，它是面向Java程序开发者的。JDK一般指Java SE JDK，使用Java开发时一般也是下载Java SE JDK。

JRE是Java Runtime Environment（Java运行时环境）的缩写，与JDK相比，它不含有Java编译器，面向最终使用Java程序的用户。

关于JDK和JRE更详细的概念和区别可以看这个网页。

JDK版本的命名

起初JDK的版本命名为JDK1.0、JDK1.1这样的格式。

在发布JDK1.2时，Sun将Java改名为Java 2，并且为了区分SE版本和EE版本与ME版本，将JDK称作J2SE（Java 2 Standard Edition），因此JDK1.2的名称变为J2SE1.2，这一命名规则持续到版本J2SE5.0（5.0和1.5.0是同一版本，5.0是产品编号，1.5.0是开发者编号）。此外，在1.2到1.4版本期间，JDK被称为SDK（Software Development Kit）。

从Java SE 6版本开始，将J2SE改称Java SE，产品编号6.0改为6，开发者编号规则不变（即1.6.0），这一规则到2020年尚未改变。

Java SE 8u31这种编号指的是Java SE 8的第31次更新，内部开发者编号为1.8.0_31，以此类推。

总结

  • Java 2基本等同于Java，并不等同于JDK1.2，是一个过时的称号，同理，J2EE等同于Java EE。
  • SDK在Java SE种也是过时的称号，它目前多指Java EE的开发工具集。
  • Java SE 6 = Java 6 = JDK 1.6，其他版本同理。

Java SE/EE/ME区别

Java是一门语言 + 一个平台，语言提供书写的语法，平台提供Java程序运行的环境。

Java有多种平台，我们经常提到的Java SE、Java EE、Java ME便是其中的3种平台。

  • Java Platform, Standard Edition (Java SE)
  • Java Platform, Enterprise Edition (Java EE)
  • Java Platform, Micro Edition (Java ME)

每一个平台都会提供一个JVM（Java Virtual Machine）和一组API。SE版本提供基础的API，此外还包括一些开发工具等，面向普通用户。EE版本建立在SE版本之上，它为大规模的应用提供更多的API（似乎试通过调用SE的API来构成新的API？）和运行时环境，面向企业用户。ME版本提供的API是SE版本API的子集，JVM也是小型的，用于嵌入式设备等小型设备的开发。此外，JDK一般指SE版本，另外两个版本的开发工具集一般不称作JDK。

总结

  • JDK一般指Java SE JDK，EE版本等其他版本一般不称作JDK。

  • 使用Java编程时一般是下载Java SE JDK（即JDK），如果需要用到Java EE，我们可以通过maven等方式引入相关的包。

整数类型（Integer Types）

整数类型有4种，分别是byte、short、int、long。Java整数类型的字节数在任何平台上都是固定的。一般情况下，我们通常使用int来表示整数，byte和short适用于数的范围比较小的应用，当数的范围比较大时，使用long。

long类型的字面量需要加上后缀l或者L，如4000000L。以0x或者0X开头表示16进制的数，如0x1C。8进制的数需要加上前缀0，如010表示十进制的8。如果不加前缀，默认为int。从Java SE 7开始，支持书写二进制，前缀为0b或者0B，如0b1001等于十进制的9，并且数字的字面量可以加下划线来帮助阅读，如1_000_000等同于1000000，编译器会自动移除下划线。

浮点数类型（Floating-Point Types）

浮点数类型有float和double，float可以保留6-7位有效数字，double可以保留15位有效数字。由于Java浮点数是以二进制的方式存储，所以在计算10进制的浮点数运算时并不是完全精确的，因为二进制无法精确表示1/10，如果需要使用精确的计算，可以用BigDecimal类。

浮点数的字面量float类型需要加上后缀f或者F，如3.14F。double的后缀为d或者D，如果不加后缀，默认为double。

浮点数有三个值表示溢出和错误。Positive infinity是正数除以0的结果，Negative infinity是负数除以0的结果，NaN (not a number)是0除以0的结果。NaN不等于任何数，判断是否是NaN需要用到函数Float.isNaN()或者Double.isNaN()。注意，整数除以0会引发异常。

字符类型（The char Type）

Java中的char和Unicode联系密切。起初，Unicode使用两个字节来表示一个字符，最多可以表示65536个字符。Java根据这时的Unicode标准设计了char数据类型，使用两个字节来表示一个字符。然而，随着汉字等表义文字字符的加入，Unicode需要表示的字符数量超出了65536，因此Java中两个字节的char类型就无法表示所有的Unicode字符了。Java SE 5.0解决了这个问题，在描述如何解决之前，需要了解几个概念。

  • 代码点（code point）：在一种编码模式中，一个字符所对应的值称为代码点。在Unicode中，代码点使用16进制表示，还要加上前缀U+。例如，在Unicode中，U+0041是拉丁字母A的代码点。Unicode中的代码点被分为17个代码平面（code planes），第一个代码平面被称作基本多文种平面（basic multilingual plane），包含经典的Unicode字符，代码点从U+0000到U+FFFF。其他16个代码平面被称作补充字符（supplementary characters），代码点从U+10000到U+10FFFF。
  • 代码单元（code unit）：代码单元是用来存储代码点的基本单位。在UTF-16中，基本多文种平面中的一个代码点只需要一个代码单元来表示，补充字符中的一个代码点需要连续的两个代码单元来表示。基本多语种平面中，有2048个未被使用的代码点（替代区域，the surrogates area），它们对应的代码单元用来表示补充字符，并且表示补充字符的代码单元也全部在这2048个代码单元之中，其中代码点U+D800到U+DBFF对应的代码单元用来表示两个连续代码单元中的前一个，代码点U+DC00到D+DFFF对应的代码单元用来表示两个连续代码单元的后一个。这样做的好处是，我们可以根据代码单元来判断它是表示一个基本多文种平面中的代码点还是一个补充字符中的代码点的前一部分还是后一部分。

在Java中，char类型用来描述UTF-16编码中的代码单元。不推荐在Java中使用char，除非需要操作UTF-16中的代码单元。

代码点和代码单元

基本多语种平面中的字符一个char对应一个代码点，补充字符中的字符两个连续的char对应一个代码点。因此，在访问字符串中的一个char时，可能访问的代码单元并不能表示一个字符，而是表示补充字符的一个代码单元，为了避免这种情况，不要使用char。

构造器

Java使用构造器来创建类的对象，构造器是一种特殊的方法，它的名字和类名相同，没有返回值类型，可以有0到多个参数，只能使用new操作符进行调用。一个类可以有1到n个构造器，如Employee类有三个构造器。构造器参数同样分为隐式参数和显式参数，同样可以重载。一个构造函数可以调用其他构造函数，必须在第一条语句调用，格式为this(参数列表)，如第三个构造器中的this(name, 1000)。

析构

Java自动回收内存，不支持析构函数，但是如果需要释放文件等非内存资源，可以使用finalize方法。

静态属性

静态属性也称为类属性，与对象属性不同，静态属性为类所有，该类的所有对象共享。同样的，如果静态属性没有显式赋值，将会被设置为默认的0值。

静态方法

静态方法可以通过类直接调用而不需要首先创建对象，也可以通过对象调用，但是一般不这么做。静态方法不能访问对象属性，也不能调用对象方法。对象方法可以访问静态属性，也可以调用静态方法。

赋值顺序

当一个构造器被调用时，对象属性会被初始化，顺序为：

  1. 对象属性被设置为默认0值；
  2. 按照声明和初始化块出现的顺序对属性进行赋值；
  3. 如果构造器第一条语句调用了其他构造函数，则执行那个构造函数；
  4. 执行构造函数。

静态属性有所不同，当类被第一次加载时静态属性就会被初始化，顺序为声明和静态初始化块出现的顺序。
