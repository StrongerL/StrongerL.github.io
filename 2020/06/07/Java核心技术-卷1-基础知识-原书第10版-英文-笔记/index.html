CH1 Java简介

Java名称由来

Java起初的名称是Oak（起这个名字的原因可能是项目的领导者之一James Gosling办公室外有一棵橡树），后来发现该名称已被另一种语言使用，就改名为了Java。一种关于命名为Java的传闻是在命名时项目组中的一人在喝名为Java的咖啡，因此起名为Java。有人为此专门采访了项目组的成员，但是大家基本上都无法详细说出命名为Java的真正原因，它出现在这次命名会议中，作为多个候选名字中的一个，最终获得通过，该采访可以在这篇文章中看到。

java发展历程

1991年，Sun公司中由Patrick Naughton和James Gosling领导的一个工程师组想要为消费者设备（如有线电视开关盒）开发一种语言，这个项目被称为Green。

1992年，Green项目发布了第一款产品，称作“*7”，然而他们的产品找不到购买的厂商。

后来该项目改名为“First Person, Inc”，由于找不到购买者，该项目在1994年解散。

与此同时，万维网也迅猛地发展着。为了展示Java的功能，Java语言开发者开发了一个浏览器——HotJava，这激发了使用Java的热潮。

Sun公司在1996年发布了第一版的Java，之后陆续发布了更多的版本。直到2009年Sun公司被Oracle公司收购，此后，Java便由Oracle公司进行维护，截止到Java8的版本发布历史可以见下图。

Java白皮书

Java的作者写了一篇很有影响力的白皮书（white paper）来介绍Java的目标和成就，他们还发布了一篇简短的概述，按照一下11个流行语（buzzwords）进行组织，可以在此处找到概述的全文。

  1. Simple
  2. Object-Oriented
  3. Distributed
  4. Robust
  5. Secure
  6. Architecture-Neutral
  7. Portable
  8. Interpreted
  9. High-Performance
  10. Multithreaded
  11. Dynamic

java applets

运行在web页面上的Java程序称为applets，它可以像图片一样嵌入在网页中，但是可以与用户进行交互。applets出现时，引起了很大的轰动。

CH2 Java编程环境

JDK相关概念

JDK与JRE区别

JDK是Java Development Kit（Java开发工具集）的缩写，它是面向Java程序开发者的。JDK一般指Java SE JDK，使用Java开发时一般也是下载Java SE JDK。

JRE是Java Runtime Environment（Java运行时环境）的缩写，与JDK相比，它不含有Java编译器，面向最终使用Java程序的用户。

关于JDK和JRE更详细的概念和区别可以看这个网页。

JDK版本的命名

起初JDK的版本命名为JDK1.0、JDK1.1这样的格式。

在发布JDK1.2时，Sun将Java改名为Java 2，并且为了区分SE版本和EE版本与ME版本，将JDK称作J2SE（Java 2 Standard Edition），因此JDK1.2的名称变为J2SE1.2，这一命名规则持续到版本J2SE5.0（5.0和1.5.0是同一版本，5.0是产品编号，1.5.0是开发者编号）。此外，在1.2到1.4版本期间，JDK被称为SDK（Software Development Kit）。

从Java SE 6版本开始，将J2SE改称Java SE，产品编号6.0改为6，开发者编号规则不变（即1.6.0），这一规则到2020年尚未改变。

Java SE 8u31这种编号指的是Java SE 8的第31次更新，内部开发者编号为1.8.0_31，以此类推。

总结

  • Java 2基本等同于Java，并不等同于JDK1.2，是一个过时的称号，同理，J2EE等同于Java EE。
  • SDK在Java SE种也是过时的称号，它目前多指Java EE的开发工具集。
  • Java SE 6 = Java 6 = JDK 1.6，其他版本同理。

Java SE/EE/ME区别

Java是一门语言 + 一个平台，语言提供书写的语法，平台提供Java程序运行的环境。

Java有多种平台，我们经常提到的Java SE、Java EE、Java ME便是其中的3种平台。

  • Java Platform, Standard Edition (Java SE)
  • Java Platform, Enterprise Edition (Java EE)
  • Java Platform, Micro Edition (Java ME)

每一个平台都会提供一个JVM（Java Virtual Machine）和一组API。SE版本提供基础的API，此外还包括一些开发工具等，面向普通用户。EE版本建立在SE版本之上，它为大规模的应用提供更多的API（似乎试通过调用SE的API来构成新的API？）和运行时环境，面向企业用户。ME版本提供的API是SE版本API的子集，JVM也是小型的，用于嵌入式设备等小型设备的开发。此外，JDK一般指SE版本，另外两个版本的开发工具集一般不称作JDK。

上述内容部分总结自该网页。

总结

  • JDK一般指Java SE JDK，EE版本等其他版本一般不称作JDK。

  • 使用Java编程时一般是下载Java SE JDK（即JDK），如果需要用到Java EE，我们可以通过maven等方式引入相关的包。

上面提到的概念是针对于Oracle JDK而言的，而实际上JDK不止一种，厂商也不止一家。关于Java还有很多的概念如JCP、JLS、JSR等，之后再补。
